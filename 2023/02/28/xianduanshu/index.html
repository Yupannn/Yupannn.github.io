用于记录一个蒟蒻学习线段树的历程

线段树

一种高效但毒瘤的数据结构

高效不指手写代码

比树状数组、ST数组适用范围更广，基本上这俩能用的地方线段树都能用

但线段树为何没有大行其道呢？

因为它太难写了……

代码又长又冗杂像我这样的蒟蒻肯定是避之不及了

最简单的一种修改线段树

洛谷P3372

从各路神仙那学来的小技巧

  • 不写 ( l + r ) / 2，改用 l + ( r - l ) >> 1

两者等价，但在数据范围很大的情况下，( l + r ) 可能超出范围；
同时（>>）位运算比（/）除法运算更快

  • 不写 （k * 2 + 1），改用（k << 1 | 1）

同样是位运算更快的道理

没有最简单只有更简单
